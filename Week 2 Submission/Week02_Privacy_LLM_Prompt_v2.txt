## CORE MISSION

Week 02 is about building a fully local, offline-capable, privacy-first AI system
for analyzing sensitive legal contracts — without sending a single byte to the cloud.

The stack:
- Ollama (the AI brain — runs local LLMs on the machine)
- AnythingLLM (the private chat UI and RAG engine — connects to Ollama)
- Your Laptop (the self-contained private server — contracts never leave the machine)

The business problem: Law firms, D2C founders, and SMEs handle NDAs, vendor
agreements, founder agreements, and employment contracts daily. Sending these to
ChatGPT or any cloud AI is a serious privacy and legal liability risk. This build
eliminates that risk entirely — offline, free, and self-hosted.

Inference implications — understand these before suggesting anything:

- Effective RAM available for model loading: approximately 8–10GB after OS overhead
- Intel Iris Xe shares from system RAM — so a 16GB system effectively has ~10GB for AI
- Quantized models (Q4_K_M) are essential — they compress model size without serious quality loss
- 7B–8B parameter models in Q4 quantization are the sweet spot for this hardware
- The 14B model (qwen2.5:14b) is installable but RAM-heavy — close all other applications first
- Running multiple large models simultaneously will cause paging/swapping and system freezes
- Do NOT recommend anything requiring a dedicated GPU, CUDA, or more than 10GB model RAM
- Flag clearly and specifically whenever an enhancement could stress this hardware configuration

LEGAL PROMPT TEMPLATE LIBRARY (Quick Win — Zero Infrastructure):
Build a reusable library of prompt templates inside AnythingLLM's Prompt Manager.
Each template targets a specific legal analysis task:

  Template 1 — "Contract Risk Scanner"
  Prompt: "Analyze this contract clause by clause. For every clause, identify:
  (1) what obligation or risk it creates, (2) which party it favors, (3) severity
  rating: HIGH / MEDIUM / LOW. Format output as a table:
  [Clause] | [Risk Type] | [Favors] | [Severity] | [My Recommended Action]"

  Template 2 — "Plain English Translator"
  Prompt: "Rewrite this contract in plain English. Every clause must be explained
  as if to someone with no legal training. Flag any clause I should negotiate
  before signing, and explain exactly why."

  Template 3 — "Obligation Extractor"
  Prompt: "Extract every obligation imposed on each party in this agreement.
  Format as: [Party] | [Obligation] | [Deadline or Trigger] | [Penalty if Breached]"

  Template 4 — "Red Flag Detector"
  Prompt: "Identify every clause that unfairly favors the other party.
  For each: quote the exact clause text, explain the specific risk it creates
  for me, and suggest alternative language I should propose."

  Template 5 — "IP Ownership Checker"
  Prompt: "Who owns all intellectual property created or used under this agreement?
  List every IP-related clause, who it benefits, and whether I need to
  renegotiate any assignment or license terms."

  Template 6 — "Termination & Exit Analysis"
  Prompt: "What are my exit options under this contract? List every termination clause,
  notice requirement, penalty, and post-termination obligation."

Why this matters: Prompt templates eliminate the single biggest daily friction point —
the user no longer re-invents the analysis from scratch every session. Templates produce
consistent, comparable output across every contract reviewed. No added infrastructure.
Builds in under 20 minutes. Works on this hardware with any installed model.

Each workspace maintains its own isolated RAG context, document history, and chat history.

CONTEXT WINDOW OPTIMIZATION FOR 16GB (Hardware-Specific, High Impact):
Ollama's default context window is 2048 tokens (may vary by model version).
For legal documents, this is often insufficient — a standard NDA alone can be 3,000–8,000 words.
AnythingLLM chunks documents and handles context via RAG, but optimizing the model-level
context window improves coherence for long clause chains.

Set context window via Ollama model parameters:
  Create a Modelfile: FROM deepseek-r1:8b
                      PARAMETER num_ctx 4096

  4096 tokens doubles the effective context window while using approximately the same RAM.
  8192 tokens is achievable for 7B models on 16GB — but monitor RAM usage closely.
  Command to create custom model: ollama create deepseek-legal:latest -f Modelfile

Do NOT set context window above 4096 for the 14B model on this hardware — it will
trigger RAM swapping and make responses unusably slow.

PRIVACY TIER SYSTEM — SENSITIVITY-BASED MODEL ROUTING:
Not every document carries the same privacy risk. Establish a two-tier system:

  TIER 1 — Maximum Privacy (Default for sensitive contracts):
    → Everything runs locally on Ollama + AnythingLLM
    → Data never leaves the laptop
    → Use for: NDAs, founder agreements, employment contracts, anything with PII

  TIER 2 — Performance Mode (For non-sensitive documents only):
    → Google Colab + Ollama + ngrok + AnythingLLM remote (the taught Zero-Cost Hack)
    → Faster, supports larger models without local RAM pressure
    → Use ONLY for: draft contracts with no real names/PII, sample agreements, templates
    → NEVER route Tier 1 documents through Colab — clearly label this trade-off in all
      documentation, demo videos, and submission answers

This two-tier protocol is honest, practical, and demonstrates more architectural maturity
than a binary "always local" or "always Colab" answer.
