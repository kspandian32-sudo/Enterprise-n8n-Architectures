Step 1 - AUDIT the v1 Build (redteam + BOTTLENECK)
Key fragile points in v1 to identify and address:
* webQueries never executed: v1 generates web search query strings but never actually fires them. The AI writes the entire document from training data + YouTube metadata alone. This is the single biggest quality gap.
* YouTube metadata only - no transcript content: GPT writes a 2,500-word document having read only video titles and descriptions. No actual expert content from videos is consumed. The AI is essentially hallucinating a document.
* Single-query research architecture: One combined queries.join(' OR ') string is sent. No sub-topic decomposition. No parallel research threads. No depth per sub-topic.
* No real web sources: The document cites no URLs, contains no article-derived content, and has no references section. It cannot be trusted or verified.
* Same AI model for all levels: GPT-4o-mini handles Complete Beginner (simple, low-stakes) and Expert Deep Dive (complex, high-quality-demanded) with identical quality ceiling.
* No delivery confirmation: Gmail fires but there is no tracking of whether the user received or engaged with the document.
* No knowledge base accumulation: Each run is isolated. Repeated research on the same topic produces duplicate docs without any cross-referencing or deduplication.
* No topic queue or scheduling: Every generation is manual. No way to queue topics for batch processing or scheduled delivery.
* Flat audit log: Google Sheets log records what was generated but offers no analytics - no topic frequency, no level distribution, no document quality scoring.
* Google Doc isolation: Generated docs live in Google Drive with no organizational structure. No folder routing by topic category or date.
